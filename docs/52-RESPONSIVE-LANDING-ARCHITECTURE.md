# Архитектура адаптивного лендинга — план рефакторинга

**Дата:** 2026-04-02  
**Связь:** детальный аудит и чеклист — [responsive-audit.md](./responsive-audit.md).

## Цель

Сохранить замысел макета: **текст в «пустой» зоне кадра, важный объект — в своей зоне**, без деформации фото при сжатии, с **единым** подходом к блокам (не разные хаки в каждой секции).

## Корневые причины текущих поломок

1. **Фиксированные размеры под «макет телефона»** — в `globals.css` для `max-width: 767px` у hero-блоков заданы `width: 367px` / `390px` и жёсткие `height: 679px` / `777px`. На реальном viewport уже 320–360px это даёт **горизонтальный скролл**, обрезку или несоотношение сторон, визуально «квадрат/ломаный кадр».
2. **Два конфликтующих источника высоты** — одновременно `aspect-ratio`, явный `height` и проценты фона; выигрывает непредсказуемое сочетание → разное поведение слайдов.
3. **Позиционирование текста через `translate` в px/rem** (`PowerBanners.tsx`, `ModelingCard` для Bridal и др.) — при изменении ширины кадра текст **не масштабируется** вместе с «пустой» областью.
4. **Смешение `object-cover` / `object-contain`** и **жёсткий `aspect-[360/259]`** у карточек специализаций без единой сетки «колонка текста / колонка изображения» на всех брейкпоинтах.

## Целевая модель (senior-подход)

| Слой | Правило |
|------|--------|
| **Контейнер сцены** | Одна ось задаёт масштаб: `width: 100%`, `max-width` из дизайна, высота из **`aspect-ratio`** или `min-height` + `overflow-hidden`, без фиксированных `367px`. |
| **Изображение** | `object-fit: cover` (или `contain` только если кадр = целый продукт без обрезки), **`object-position`** в процентах — якорь важного объекта; без растягивания (`object-fill`). |
| **Текст** | В пределах **padding + flex/grid** с долями колонок (`fr` / `%`), без отрицательных `translate` как основного инструмента; опционально лёгкий `clamp()` для типографики. |
| **Токены** | CSS-переменные для высот hero и сдвигов по брейкпоинтам (`--section2-block-visible-height` и т.д.) — одно место правды в `globals.css`. |

## Фазы работ

| Фаза | Область | Действия |
|------|---------|----------|
| **A** | Hero (`PowerBanners` + `globals.css`) | Убрать фиксированные ширины mobile; единая пропорция блока; упростить mobile-фон section1/2 (без `114%` / `margin-left: -7%` где возможно). |
| **B** | Карточки специализаций (`ModelingCard` + `ModelingBlock*`) | Вынести повторяющиеся отступы в сетку; заменить крупные `-translate-x-*` на проценты/`clamp` или колонки; разрешить перенос строк на узких экранах. |
| **C** | Остальные секции | По [responsive-audit.md](./responsive-audit.md): галерея, нав, заголовки секций. |

## Критерии готовности

- Нет горизонтального скролла на 320px (кроме намеренных каруселей).
- Фото без растягивания; при сужении — масштаб и кроп предсказуемы.
- Текст hero и карточек не уезжает на произвольные пиксели относительно кадра.
